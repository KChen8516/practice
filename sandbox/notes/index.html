Variables

let and const are both supported by modern browsers and have better use cases due to their block level scoping

The const keyword

  • const must be assigned a value prior to run time.
  • a const variable can be mutated but not reassigned (e.g. changing obj key/values, mutating arrays)

Data Types

Primitive

  • String
  • Number
  • Boolean
  • Null (JavaScript bug shows typeof as object)
  • Undefined
  • Symbols (A new ES6 data type)

Reference

  • Arrays
  • Object Literals
  • Functions
  • Dates
  • Anything else...

Language Idiosyncracies

  • JS will do some automatic type conversion such as '5' + 6 = '56'
  • When calling sort on an array of numbers, it'll only sort the first digit. You need to provide a compare function.
  • When using a weaker comparison operator == JS will ignore data types when comparing (100 == '100' // true)
  • Using the === operator will check both the value and the type

ES6 Features

  • Template literals are capable of rendering function results and ternary operators!
  • Use the FOR IN loop to get key value pairs in an object literal

Immediately Invocable Function Expressions (IIFE)

  • Declared and run at the same time
  • Not useful combined with module patterns
